### 1.1 Random Forest (RF) — `train_predict_rf()`
- **ที่มา:** อัลกอริทึม Machine Learning ตระกูล Ensemble (Leo Breiman, 2001) สร้างต้นไม้ตัดสินใจ (Decision Tree) หลายต้นแล้วเฉลี่ยผล
- **Input (Features) 4 ตัว:**
  - `Return` = เปอร์เซ็นต์การเปลี่ยนแปลงราคาวันนี้ = (ราคาปิดวันนี้ − เมื่อวาน) / เมื่อวาน
  - `SMA_10` = ค่าเฉลี่ยราคาปิด 10 วัน (เส้นค่าเฉลี่ยระยะสั้น)
  - `SMA_50` = ค่าเฉลี่ยราคาปิด 50 วัน (เส้นค่าเฉลี่ยระยะกลาง)
  - `Volatility` = ส่วนเบี่ยงเบนมาตรฐานของ Return ย้อนหลัง 20 วัน (ความผันผวน)
- **Target:** `Target_Return` = Return ของ "วันถัดไป" (เลื่อนข้อมูลขึ้น 1 แถว)
- **วิธีคำนวณ:** เทรนด้วยต้นไม้ 100 ต้น (`n_estimators=100`) จากข้อมูลย้อนหลัง 5 ปี → ทำนายเป็น `% การเปลี่ยนแปลงของวันพรุ่งนี้`
- **โค้ดอ้างอิง:** `daily_predictor.py:50`

### 1.2 ARIMA — `train_predict_arima()`
- **ที่มา:** AutoRegressive Integrated Moving Average (Box & Jenkins, 1970) — โมเดลสถิติคลาสสิกสำหรับอนุกรมเวลา (Time Series)
- **พารามิเตอร์ที่ใช้:** `order=(5, 1, 0)`
  - `p=5` → ใช้ราคา 5 วันก่อนหน้ามาพยากรณ์ (AutoRegressive)
  - `d=1` → หาผลต่าง 1 ครั้งเพื่อทำให้ข้อมูลนิ่ง (Stationary)
  - `q=0` → ไม่ใช้ Moving Average term
- **วิธีคำนวณ:** พยากรณ์ราคาปิดวันพรุ่งนี้ (`forecast(steps=1)`) แล้วแปลงเป็น % = (ราคาที่ทำนาย − ราคาปิดล่าสุด) / ราคาปิดล่าสุด
- **โค้ดอ้างอิง:** `daily_predictor.py:57`

### 1.3 Weighted Ensemble + Adaptive Feedback — `compute_model_weights()`
- **ที่มา:** แนวคิด Ensemble Averaging + Online Learning (ปรับน้ำหนักตามผลงานจริง)
- **สูตรรวม:**
  ```
  ensemble = rf_pct × w_rf + arima_pct × w_arima
  ทิศทาง = "Up" ถ้า ensemble > 0, ไม่งั้น "Down"
  ```
- **การคำนวณน้ำหนัก (w):** ดูประวัติย้อนหลังว่าแต่ละโมเดลทายทิศทาง "ถูก" กี่ครั้ง แล้วใช้ **Laplace Smoothing**:
  ```
  rf_acc    = (จำนวนที่ RF ทายถูก + 1) / (จำนวนครั้งทั้งหมด + 2)
  arima_acc = (จำนวนที่ ARIMA ทายถูก + 1) / (จำนวนครั้งทั้งหมด + 2)
  w_rf      = rf_acc / (rf_acc + arima_acc)
  w_arima   = arima_acc / (rf_acc + arima_acc)
  ```
- **ทำไมต้อง +1/+2 (Laplace):** กันไม่ให้โมเดลใดถูกปิดเสียง (น้ำหนัก 0) ตอนข้อมูลยังน้อย และถ้าไม่มีประวัติเลยจะเริ่มที่ 50/50
- **โค้ดอ้างอิง:** `daily_predictor.py:133` และการนำไปใช้ที่ `daily_predictor.py:372`

### 1.4 News Sentiment (สูตรที่ 4 — เพิ่มใหม่) — `train_predict_news()`
- **ที่มา:** Sentiment Analysis / Natural Language Processing — แนวคิดว่า "อารมณ์ของข่าว" นำหน้าการเคลื่อนไหวของราคา (event-driven). เริ่มต้นด้วย **VADER** (Hutto & Gilbert, 2014) ซึ่งเป็น lexicon-based sentiment ที่ออกแบบมาเหมาะกับข้อความสั้นอย่างพาดหัวข่าว
- **Input:** พาดหัว/สรุปข่าวจากหลายแหล่ง (RSS การเงินทั่วโลก + ข่าวรายหุ้นจาก `yfinance.news`) ที่เกี่ยวข้องกับแต่ละ symbol
- **⭐ เงื่อนไขบังคับ — กรองความน่าเชื่อถือของแหล่งข่าว (Source Credibility ≥ 90%):**
  - รับข่าว **เฉพาะจากแหล่งที่มีคะแนนความน่าเชื่อถือ ≥ 90 เท่านั้น** ข่าวจากแหล่งที่ต่ำกว่าจะถูกทิ้งทั้งหมดก่อนนำไปวิเคราะห์
  - ใช้ **whitelist แหล่งข่าวที่ผ่านการให้คะแนน** (อ้างอิงมาตรฐานสากล เช่น NewsGuard, Media Bias/Fact Check) เก็บเป็นตารางในโค้ด `SOURCE_CREDIBILITY = {domain: score}`
  - ตัวอย่างแหล่งที่ผ่าน (≥90): Reuters (100), Associated Press (100), Bloomberg (95), Wall Street Journal (95), Financial Times (95), CNBC (92), MarketWatch (90)
  - แหล่งที่ไม่อยู่ใน whitelist ถือว่า "ไม่ผ่าน" โดยปริยาย (ปลอดภัยไว้ก่อน)
  - แสดงบน UI ว่าข่าวที่ใช้มาจากกี่แหล่ง และคะแนนความน่าเชื่อถือเฉลี่ย
- **วิธีคำนวณ (เป็นขั้นตอน):**
  1. **กรองแหล่ง:** เก็บเฉพาะข่าวที่ `SOURCE_CREDIBILITY[domain] ≥ 90` (ขั้นตอนนี้ทำก่อนเสมอ)
  2. ให้คะแนนอารมณ์แต่ละข่าวที่ผ่าน → `compound score` อยู่ในช่วง −1 (ลบมาก) ถึง +1 (บวกมาก)
  3. เฉลี่ยคะแนนของข่าวที่เกี่ยวกับ symbol นั้น **ถ่วงน้ำหนักตามคะแนนความน่าเชื่อถือของแหล่ง** (แหล่ง 100 มีน้ำหนักมากกว่าแหล่ง 90) และตามความใหม่ของข่าว
  4. แปลงเป็น % คาดการณ์ราคา:
     ```
     news_score (%) = avg_sentiment × k
     ```
     โดย `k` = ตัวคูณปรับสเกล (calibration factor) ให้ขนาดใกล้เคียงกับ rf_pct/arima_pct
  4. ทิศทาง = "Up" ถ้า news_score > 0, ไม่งั้น "Down"; ถ้าไม่มีข่าวที่เกี่ยวข้องให้คืนค่า 0 (เป็นกลาง)
- **ออปชันขั้นสูง:** แทน VADER ด้วย **Claude API** วิเคราะห์ข่าวเชิงลึกแล้วคืนค่า JSON sentiment (แม่นกว่ามากกับข่าวเศรษฐกิจที่กำกวม แต่มีค่าใช้จ่าย/ต้องมี API key)
- **ไฟล์ที่จะเพิ่ม:** `news_analyzer.py` (เก็บข่าว + ให้คะแนน) เรียกใช้จาก `daily_predictor.py`
- **Dependency ใหม่:** `vaderSentiment` ใน `requirements.txt`

## 🔗 ส่วนที่ 3 — ถ้ารวมทุกสูตรมาคำนวณพร้อมกัน ผลลัพธ์จะเป็นยังไง

- **สูตรที่ 3 (Ensemble) ทำหน้าที่เป็น "ตัวรวม"** ของสูตรอินพุตทั้ง 3 ตัว คือ สูตรที่ 1 (RF), 2 (ARIMA) และ 4 (News)
- ระบบจะ **ถ่วงน้ำหนักตามผลงานจริงในอดีต** (ไม่ใช่เฉลี่ยเท่ากันแบบตายตัว) — สูตรไหนแม่นกว่าจะมีสิทธิ์ออกเสียงมากกว่าในรอบถัดไป
- สูตรรวมหลังเพิ่มข่าว (เป้าหมายใหม่) — **ถ่วงน้ำหนัก 3 ทาง:**
  ```
  ensemble = rf_pct×w_rf + arima_pct×w_arima + news_score×w_news
  โดย   w_rf + w_arima + w_news = 1   (ผลรวมน้ำหนักเท่ากับ 100%)
  ```
- **ข้อดีของการรวม:** ลดความเสี่ยงจากการพึ่งโมเดลเดียว, ถ้าโมเดลหนึ่งพลาดอีกตัวช่วยถ่วง, และระบบจะ "เรียนรู้" ไปเรื่อยๆ ว่าในแต่ละหุ้น/สินทรัพย์ ควรเชื่อใครมากกว่า
- **ผลลัพธ์ที่คาดหวัง:** ความแม่นยำโดยรวม (Overall Accuracy) ควรนิ่งและสูงกว่าการใช้สูตรเดียว เมื่อสะสมข้อมูลมากพอ

## 📰 ส่วนที่ 6 — ระบบคำนวณใหม่จากข่าวสาร (News Sentiment Prediction)

**เป้าหมาย:** รวบรวมข่าวจากอินเทอร์เน็ตให้ได้มากที่สุดเท่าที่เข้าถึงได้ → วิเคราะห์อารมณ์ (Sentiment) → แปลงเป็นสัญญาณทำนาย

**สถาปัตยกรรมที่เสนอ (ไฟล์ใหม่ `news_analyzer.py`):**
1. **เก็บข่าว (Collect):** ขยายจาก RSS 2 แหล่งเดิม → เพิ่มหลายแหล่ง
   - การเงินทั่วโลก: CNBC, WSJ, Reuters, Yahoo Finance, MarketWatch, Bloomberg RSS
   - รายหุ้น: ใช้ `yfinance` `.news` ของแต่ละ symbol (ข่าวเฉพาะตัว)
   - ไทย: ถ้าต้องการ เพิ่ม RSS ไทย (กรุงเทพธุรกิจ/ฐานเศรษฐกิจ)
2. **⭐ กรองความน่าเชื่อถือ (Filter ≥ 90%) — ขั้นตอนใหม่บังคับ:**
   - ดึง domain ของแต่ละข่าว → เทียบกับตาราง `SOURCE_CREDIBILITY`
   - **ทิ้งทุกข่าวที่แหล่งมีคะแนน < 90 หรือไม่อยู่ใน whitelist**
   - เก็บ log ว่ากรองทิ้งไปกี่ข่าว / เหลือกี่ข่าว เพื่อความโปร่งใส
4. **วิเคราะห์อารมณ์ (Analyze):** เริ่มด้วย **VADER** (lexicon ฟรี เหมาะกับพาดหัวข่าว) ให้คะแนน −1..+1 เฉพาะข่าวที่ผ่านการกรองในข้อ 2 แล้วเฉลี่ย/ถ่วงน้ำหนักตามคะแนนความน่าเชื่อถือ + ความเกี่ยวข้องกับแต่ละ symbol
   - *(ออปชันขั้นสูง: ใช้ Claude API วิเคราะห์ข่าวเชิงลึกเป็น JSON sentiment — แม่นกว่าแต่มีค่าใช้จ่าย)*
5. **แปลงเป็นสัญญาณ:** `news_score` (% คาดการณ์จากข่าว) เข้าสูตร ensemble ในส่วนที่ 3
6. **Dependency ใหม่:** เพิ่ม `vaderSentiment` ใน `requirements.txt`

**ผลลัพธ์:** การ์ดแต่ละตัวจะมีบรรทัด "มุมมองจากข่าว: 📈 บวก / 📉 ลบ / ➖ กลาง" พร้อมจำนวนข่าวที่อ้างอิง **และระบุว่ามาจากแหล่งน่าเชื่อถือ ≥90% เท่านั้น**

## ♻️ ส่วนที่ 7 — สะสมผลลัพธ์เพื่อใช้คำนวณรอบถัดไป (Self-Improving Loop)

**สถานะปัจจุบัน:** ระบบ feedback loop มีอยู่แล้วสำหรับ RF/ARIMA (`compute_model_weights`) — เก็บประวัติใน `prediction_history.json` และปรับน้ำหนักทุกวัน ✅

**สิ่งที่จะเพิ่ม:**
1. **ขยาย feedback loop ให้ครอบคลุม News** — เก็บ `news_pct` ลงในประวัติเหมือน `rf_pct`/`arima_pct` แล้วให้ระบบเรียนรู้ว่าข่าวเชื่อถือได้แค่ไหนต่อ symbol → ได้ `w_news` อัตโนมัติ (3-way weighting)
2. **เก็บ Log ความแม่นยำแยกตามวิธี** — ตารางใหม่: "RF แม่น X% / ARIMA Y% / News Z% / Ensemble W%" เพื่อพิสูจน์ว่าการรวมดีกว่าจริง
3. **เลือกสูตรที่แม่นที่สุดอัตโนมัติ** — ถ้าวิธีใดแม่นนำลิ่วต่อเนื่อง ระบบจะเพิ่มน้ำหนักให้เด่นขึ้น (มี mechanism อยู่แล้ว ขยายเป็น 3 ทาง)
4. **กันข้อมูลเพี้ยน** — คงตรรกะข้ามวันหยุด/ราคาขาด (มีอยู่แล้วที่ `daily_predictor.py:315`)

> หลักการ: ทุกครั้งที่ทายแล้วรู้ผลจริง → บันทึก → ปรับน้ำหนัก → ใช้รอบถัดไป เป็นวงจรไม่รู้จบเพื่อไล่หาความแม่นยำสูงสุด
